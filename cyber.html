الفصل الأول • التشفير بمفتاح عام (Public-Key Cryptography)

لماذا نحتاج هذا النوع من التشفير؟

التشفير المتماثل (مفتاح واحد) يعاني من مشكلتين كبيرتين:

1. تبادل المفتاح: يجب مشاركة المفتاح السري بين الطرفين (خطر السرقة).
2. التوقيعات الرقمية: كيف نثبت أن الرسالة جاءت فعلاً من شخص معين؟

Why Do We Need Public-Key Cryptography?

Symmetric encryption (single key) has two major problems:

1. Key exchange: Both parties must share the secret key (risk of theft).
2. Digital signatures: How to prove the message really came from a specific person?

الحل السحري: زوج من المفاتيح

  • كل شخص يولّد مفتاحين:
    • مفتاح عام (Public Key) → يُنشر للجميع (مثل رقم الهاتف)
    • مفتاح خاص (Private Key) → يبقى سرياً عند صاحبه فقط
  • المفتاحان مرتبطان رياضياً (مثل RSA).

The Magic Solution: A Pair of Keys

  • Every person generates two keys:
    • Public Key → Published for everyone (like a phone number)
    • Private Key → Kept completely secret by the owner
  • The two keys are mathematically linked (e.g., RSA).

كيفية التشفير (السرية فقط)

أحمد يريد إرسال رسالة سرية لسارة:

يستخدم مفتاح سارة العام للتشفير
سارة فقط تستطيع فك التشفير بمفتاحها الخاص

How Encryption Works (Confidentiality Only)

Ahmed wants to send a secret message to Sara:

He encrypts using Sara’s public key
Only Sara can decrypt it with her private key

كيفية التوقيع الرقمي (التحقق من الهوية)

أحمد يحسب "بصمة" الرسالة (الهاش) ثم يشفرها بمفتاحه الخاص
أي شخص يستطيع التحقق بـ مفتاح أحمد العام

Digital Signature (Authentication)

Ahmed calculates the “fingerprint” (hash) of the message then encrypts it with his private key
Anyone can verify using Ahmed’s public key

الجمع بين السرية + التوقيع

أحمد يوقّع أولاً بمفتاحه الخاص، ثم يشفر النتيجة بمفتاح سارة العام.

Combining Confidentiality + Authentication

Ahmed signs first with his private key, then encrypts the result with Sara’s public key.

مزايا مذهلة

  • المفتاح الخاص لا يُرسل أبداً
  • يمكن تغيير المفاتيح في أي وقت
  • كل شخص يولّد مفاتيحه بنفسه (آمن جداً)

Amazing Advantages

  • Private key is never sent
  • Keys can be changed at any time
  • Everyone generates their own keys locally (very secure)
الفصل الثاني • الدوال الهاش التشفيرية (Cryptographic Hash Functions)

ما هي دالة الهاش؟

تأخذ أي نص أو ملف (طوله متغير) وتعطيك رقم ثابت الطول (بصمة).

مثال: ملف 10 جيجا → يصبح 64 حرف فقط (SHA-512)
خاصية ذهبية: تغيير حرف واحد = تغيير كامل في الهاش (مستحيل التنبؤ)

What is a Hash Function?

It takes any text or file (variable length) and produces a fixed-size fingerprint.

Example: 10 GB file → becomes only 64 characters (SHA-512)
Golden Property: Changing one letter = completely different hash (impossible to predict)

مقارنة أشهر الدوال (SHA)

الدالة حجم الهاش حجم البلوك عدد الخطوات
SHA-1 160 بت 512 بت 80
SHA-256 256 بت 512 بت 64
SHA-512 512 بت 1024 بت 80

SHA Family Comparison

Function Hash Size Block Size Steps
SHA-1 160 bits 512 bits 80
SHA-256 256 bits 512 bits 64
SHA-512 512 bits 1024 bits 80

تطبيقات الدالة الهاش

  • مصادقة الرسالة (نضيف "ملح" Salt لمنع التلاعب)
  • التوقيع الرقمي (نحسب الهاش أولاً ثم نشفره فقط — أسرع بكثير)
  • تخزين كلمات المرور بأمان
  • التحقق من سلامة الملفات

Applications of Hash Functions

  • Message Authentication (add Salt to prevent tampering)
  • Digital Signature (calculate hash first, then sign only the hash — much faster)
  • Secure password storage
  • File integrity verification
التطبيقات العملية • Practical Applications
1. السرية فقط: تشفير الرسالة بمفتاح المستقبل العام

2. التحقق فقط: توقيع بمفتاح المرسل الخاص

3. الاثنان معاً: أقوى مستوى أمان

Practical Applications

1. Confidentiality Only: Encrypt with receiver’s public key

2. Authentication Only: Sign with sender’s private key

3. Both Together: Highest security level
أسئلة وإجابات للمراجعة السريعة • Quick Review Questions & Answers
1. ما هي مشكلتا التشفير المتماثل؟
تبادل المفتاح + صعوبة التوقيعات الرقمية
1. What are the two problems of symmetric encryption?
Key exchange + Digital signatures difficulty
2. كيف يعمل التشفير لضمان السرية فقط؟
يشفر المرسل بمفتاح المستقبل العام، والمستقبل يفك بمفتاحه الخاص
2. How does encryption ensure confidentiality only?
Sender encrypts with receiver’s public key, receiver decrypts with private key
3. ما هي خاصية دالة الهاش الأهم؟
تغيير حرف واحد يغير الهاش كلياً (مستحيل التنبؤ)
3. What is the most important property of a hash function?
Changing one character completely changes the hash (unpredictable)
4. لماذا نستخدم الهاش في التوقيع الرقمي؟
لأنه سريع جداً — نكتفي بتشفير 64 حرف فقط بدلاً من الملف كله
4. Why do we use hash in digital signatures?
It is very fast — we only encrypt 64 characters instead of the whole file
5. ما الفرق بين SHA-256 وSHA-512؟
SHA-512 أكثر أماناً (512 بت) ويستخدم بلوك أكبر
5. What is the difference between SHA-256 and SHA-512?
SHA-512 is more secure (512 bits) and uses larger blocks
